A small commentary:
The main idea of this homework is to remind ourselves a way to use forms on the webpages.
This is a basic way to interact with the user. This form requires at least two fields to be filled out: E-mail and Password. Others aren't necessary, so user may ignore them.
This task improved knowledge about labels, as the basic lesson had nothing to say about adding labels with checkbox and radio type inputs. Correct usage requires us to use ID attribute of the <input> tag, then refer to it with "for" attribute of a <label> tag. In this case <label> area becomes responsive to clicks, and also check/unchecks it's input.
Next trouble was about using button to redirect user to another page.
First of all I've tried to it with wrapping <a> tag into <button> tag. But it resulted in printing a link looking text on the button. Few tries resulted into wrapping them vice versa. Think this is the propper way to use a button as a link.
Question remains, if I were to offer the user to go to a particular section, I'd added an ID to a div/section/w.e., but this results in adding a #id_value to the URL. Does it harm to SEO and am I still at the same page after URL modification?
