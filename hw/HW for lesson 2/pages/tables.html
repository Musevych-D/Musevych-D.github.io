Lesson practise with tables

Colspan

There is an issue about making a "mirror" merging of cells. Like, when merging cells changes the quatity of td tags needed to build a table, it will result in inapropriate table look. For example, we have a 3x3 table. Merging 1+2, 4-6 and 8-9 cells would result in autochanging table to 3x2 one. To prevent that, at least one element should "hold" a table structure, in other words, having all needed cells in it's row. If we had a fourth row with 3 cells, such combining would be looking the same as we wanted, otherwise, table would rebuilt itself.
